## Overview

Static, build-free **single-page** site (HTML + CSS + vanilla JS), Vercel-ready as-is. One page,
`index.html`, holds every section; navigation is in-page anchors. ALMA = studios-only building in
**Gastown, Vancouver**; pet-free; keyless; move-in promotions. (The earlier `units.html` was
merged into `index.html` and deleted, per the owner's single-website requirement.)

## Hub / god nodes

1. **`styles.css`** — highest blast radius; the `:root` token block drives the whole page.
2. **`units.js` `UNITS` array** — single source of truth for availability, prices, video IDs
   **and** the `ItemList` SEO schema; cards + structured data update together.
3. **`index.html`** — the only page; nav/footer/anchors all resolve within it (no cross-page risk).

## Design rationale

- Single-page static site (owner wanted one website, not separate HTML pages). No build step.
- Unit cards are intentionally minimal: number, price, Apply, Book a tour (and Watch video once a
  `videoId` exists). Thumbnails auto-derive from the video ID; no manual unit photos.
- Promotions are a visual showcase (two image tiles + "certain units / conditions apply"); not
  tied to named units in the public UI.
- Apply = `mailto` with subject `ALMA <num> - Rental Application`; tour = Calendly popup.

## Design system notes (latest pass)

- **Fixed a real layout bug:** `.nav-mobile` had no base `display:none`, so the mobile menu
  rendered as a plain unstyled link row on desktop. Now hidden by default; shown only in the
  mobile breakpoint. The desktop nav is a framed **pill group**.
- **Button styles** (reusable, ALMA palette, reduced-motion guarded): `.btn-lift` (colored
  shadow lift) and `.btn-ripple` (expanding blue circles; needs 5 empty `<span>`s + 1 label
  span). Featured on the contact band.
- **Unit cards** use a floating gradient header (negative-margin media + colored shadow).
- **Video modal is portrait (9:16)** to fit the vertical YouTube tour videos. Unit 665 has the
  first real `videoId` (`AuJsVOf0lS0`).
- **Footer social:** Instagram (`@thisismyalma`) + WhatsApp (`+1 604 213 1171`) only.
- **Two more layout bugs fixed:** `.hero-inner` used `max-width: 18ch` (ch on the small
  container font ≈ 150px, squeezing the hero); now `min(100%, 60rem)` and fully responsive.
  Nav CTA used a non-existent `--sp-5` token (padding collapsed, text spilled); fixed. Footer
  column links were `inline-block` (ran together like a sentence); now `block`.
- **Named button components** (ALMA palette, reduced-motion + touch safe):
  `.btn-apply` (navy base, orange sweep fill — all Apply-now actions) and `.btn-tour`
  (dark, inverts to outline on hover, calendar icon — all Book-a-tour actions, with an
  `.on-dark` variant for the hero). Unit cards stack these full-width; "Watch video" removed
  (the media play button is the only video entry point).
- **Promotions** use flip cards (Uiverse/Dailleer) with `@media (hover:none)` fallback so the
  details show on touch devices.
- **Button hierarchy:** Apply-now is the **orange** primary CTA; Book-a-tour is **dark ink**
  secondary, so they are clearly distinct.
- **Currency switcher** (`units.js`): converts every `.money` span from CAD to USD/EUR/GBP/BRL/
  **KRW** live via the Frankfurter API (`api.frankfurter.app`, ECB data), cached after first
  fetch, re-applied after each sort, and **degrades gracefully to CAD** if the network/API is
  unavailable. Prices use `Intl.NumberFormat("en-US")` so CAD shows as `CA$` (distinct from USD `$`).
  New external dependency: `api.frankfurter.app` (no key, CORS-enabled).
